# SPEC.md — color-conv

## サイト概要

- サイト名: カラー変換ツール（仮）
- 目的: 色コードの相互変換と配色生成をブラウザ上で完結させる
- 収益モデル: AdSense + デザインツール系アフィリエイト（Canva、Adobe等）+ A8.net
- 技術構成: React 19 + TypeScript + Vite 8
- デプロイ: Cloudflare Pages + GitHub自動デプロイ
- 外部データ更新不要。色の計算は数学的に確定しているのでメンテナンスフリー

## フェーズ分け

### Phase 1（初期リリース）
1. カラーコード変換（メイン）
2. 配色ジェネレーター
3. グラデーション生成
4. コントラスト比チェッカー

### Phase 2
5. 色覚シミュレーション（表示のみ。警告機能はPhase 3以降）
6. 和色・慣用色名辞典
7. 独自カラーピッカー（色相環 + 彩度/明度パネル）

## ツール間の色引き継ぎ

URLクエリパラメータで実現する。
- 例: `/palette?base=FF5733`
- 各ツールで選択/入力した色から他ツールへのリンクを提供
- localStorageでも直近の色を保持（`color-conv:lastColor`）

## ツール1: カラーコード変換（`/`）

### 入力
どれか1つを入力すると他がすべて連動:
- HEX（#FF5733）
- RGB（255, 87, 51）
- HSL（14°, 100%, 60%）
- HSV / HSB（14°, 80%, 100%）
- CMYK（0%, 66%, 80%, 0%）
- カラーピッカー（Phase 1ではブラウザネイティブ `<input type="color">` + HEX直接入力）

### 出力
- 全形式の値を同時表示
- 各値の横にコピーボタン
- プレビュー（大きめの色見本）
- CSS表記（`color: #FF5733;`、`color: rgb(255, 87, 51);` 等）もコピー可能

## ツール2: 配色ジェネレーター（`/palette`）

### 入力
ベースカラー1色（ツール1から引き継ぎ or 直接指定）

### 出力（配色パターンごとにタブ切り替え）
- 補色（Complementary）
- 類似色（Analogous）
- トライアド（Triadic）
- スプリットコンプリメンタリー（Split-Complementary）
- テトラード（Tetradic / Square）
- モノクロマティック（明度・彩度の段階）

### 各色の表示
- 色見本 + HEX + RGB
- コピーボタン
- 全色をまとめてCSS変数としてコピーする機能

## ツール3: グラデーション生成（`/gradient`）

### 入力
- 開始色・終了色（2色。追加で中間色も設定可能）
- 方向（線形: 角度指定 / 放射状）
- ステップ数（2〜20）

### 出力
- グラデーションプレビュー（大きめの帯）
- 各ステップの色コード一覧
- CSSの `linear-gradient()` / `radial-gradient()` コードをコピー可能

## ツール4: コントラスト比チェッカー（`/contrast`）

### 入力
前景色（テキスト色）と背景色

### 出力
- コントラスト比（例: 4.5:1）
- WCAG 2.1 適合レベルの判定:
  - AA 通常テキスト（4.5:1以上）
  - AA 大テキスト（3:1以上）
  - AAA 通常テキスト（7:1以上）
  - AAA 大テキスト（4.5:1以上）
- プレビュー（実際にその色の組み合わせでテキストを表示）
- 不合格の場合、合格ラインに近い色を提案

## ツール5: 色覚シミュレーション（`/vision`）[Phase 2]

### 入力
任意の色（1色 or 配色セット）

### 出力
- 一般色覚（C型）
- 1型2色覚（P型 / 赤が見えにくい）
- 2型2色覚（D型 / 緑が見えにくい）
- 3型2色覚（T型 / 青が見えにくい）
- モノクロ（グレースケール）

各タイプでの見え方を色見本で並べて表示。

### Phase 3以降
配色セット入力時の「この配色はP型で区別しにくい」等の警告機能。

## ツール6: 和色・慣用色名辞典（`/colors`）[Phase 2]

- JIS慣用色名（269色）と和色名のデータ（JSON、約30KB）
- 入力した色に最も近い慣用色名を表示（CIEDE2000で判定）
- 一覧ページで色名から検索・HEXを取得

## 色変換ロジック

すべてクライアントサイドの数学演算で完結。

### RGB <-> HEX
単純な16進変換。

### RGB <-> HSL
```
R, G, B を 0〜1 に正規化
max = max(R, G, B), min = min(R, G, B)
L = (max + min) / 2
S = (max - min) / (1 - |2L - 1|)  ※L≠0, L≠1のとき
H = 色相角（R, G, B の大小関係で場合分け）
```

### RGB -> CMYK
```
K = 1 - max(R, G, B)
C = (1 - R - K) / (1 - K)
M = (1 - G - K) / (1 - K)
Y = (1 - B - K) / (1 - K)
```
**注記必須**: CMYKはICCプロファイルを考慮しない概算であり、実際の印刷色とは異なる。

### コントラスト比（WCAG 2.1）
```
相対輝度 = 0.2126 * R' + 0.7152 * G' + 0.0722 * B'
（R', G', B' はガンマ補正を線形化した値）
コントラスト比 = (L1 + 0.05) / (L2 + 0.05)
```

### 最近接色判定（CIEDE2000）
RGB → Lab → CIEDE2000 距離で判定。CIE76より知覚的に正確。
RGB → XYZ → Lab の変換パイプラインを実装する。

### 色覚シミュレーション [Phase 2]
Brettelらの変換行列を使用。RGB → LMS色空間に変換し、各色覚タイプの射影行列を適用してRGBに戻す。

## 差別化

- 変換・配色・コントラスト・色覚シミュレーションが1サイトにまとまっている
- 全ツール間でURLクエリパラメータを通じて色を引き継げる
- 和色辞典は海外ツールにない独自コンテンツ（Phase 2）
- CSSコードのコピー機能（開発者がそのまま使える）
- 色覚シミュレーション対応は日本語ツールではかなり少ない（Phase 2）

## 免責事項

- CMYKの変換はICCプロファイルを考慮しない概算であり、実際の印刷色とは異なる
- 色覚シミュレーションは学術的な変換モデルに基づく近似であり、個人差がある
- モニターの表示特性により、実際に見える色は環境によって異なる
